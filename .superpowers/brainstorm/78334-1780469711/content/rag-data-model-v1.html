RAG 数据模型草图

第一版使用 PostgreSQL 保存可追踪元数据，Milvus 保存向量检索索引。

PostgreSQL
rag_documents
id, source_uri, title, content_hash, mime_type, status, metadata_json, created_at
rag_chunks
id, document_id, chunk_index, content, heading_path, token_count, char_start, char_end, metadata_json
rag_ingestion_runs
id, document_id, status, error_message, started_at, finished_at
Milvus
hello_agents_rag_chunks
id = chunk_id
vector
document_id
source_uri
metadata filters
检索命中 chunk_id 后
回 PostgreSQL 取完整 chunk 文本和来源信息
Why This Shape

优点

  • chunk 文本、标题路径、来源可追踪
  • Milvus 只承担向量搜索，schema 简单
  • 支持后续按 document_id 重建索引或删除文档

暂不做

  • 第一版不引入 Neo4j 知识图谱
  • 第一版不做多 collection 分库
  • 第一版不做复杂权限模型
