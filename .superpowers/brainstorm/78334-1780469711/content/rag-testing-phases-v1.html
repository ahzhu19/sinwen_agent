测试策略与分阶段实施

先把可测试的核心链路跑通，再逐步加查询增强、删除重建和 Agent 集成。

Implementation Phases
Phase 1
模型 / 存储 / MarkItDown 转换
Phase 2
Markdown 分块 / embedding / Milvus 写入
Phase 3
search / answer / sources
Phase 4
delete / stats / reindex / Agent 工具接入
Testing Pyramid

必须覆盖

  • Markdown 分块：标题路径、overlap、表格/代码块保留
  • 摄取事务：转换成功、embedding 失败、Milvus 失败
  • 检索回填：Milvus 命中 chunk_id 后取回正文和来源
  • RagTool：ingest/search/answer 的输出格式

可用 Fake 替代

  • FakeConverter 代替 MarkItDown
  • FakeEmbeddingProvider 代替真实 API
  • FakeVectorStore 代替 Milvus
  • FakeLLM 代替真实生成
MVP Definition
给定一个 Markdown/PDF/Word 文件，执行 ingest 后可以 search 到相关 chunk，并用 answer 生成带来源编号的回答。
